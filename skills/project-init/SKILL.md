# project-init — оркестратор старта нового проекта

Это skill вызывается командой `/dm-cc-assistant:project-init`. Он запускает четыре subagent'а последовательно через Task tool. Все файлы создаются в текущей рабочей директории (cwd).

## Твоя роль как основного Claude

Ты — дирижёр. Сам интервью не ведёшь и документы не пишешь. Твоя работа:
1. Коротко объяснить пользователю что будет происходить.
2. Запускать агентов по порядку через Task tool.
3. После каждого агента проверять что ожидаемый файл создан.
4. Останавливать весь процесс если агент не создал файл или пользователь отказался подтверждать превью.

## Вводное сообщение пользователю

В начале выведи короткое сообщение на русском:

> Запускаю `project-init`. Будет 4 шага:
> 1. Интервью по продукту → `OVERVIEW.md`
> 2. Интервью по архитектуре → `ARCHITECTURE.md`
> 3. Генерация `CLAUDE.md` из первых двух
> 4. Скаффолдинг `.claude/` (только для KMP в v1)
>
> На каждом шаге ты увидишь превью и сможешь подтвердить или отказаться. Работаем в текущей директории.

## Шаг 1 — OVERVIEW.md

Запусти Task с `subagent_type: overview-interviewer`. В prompt передай:

> Проведи интервью с пользователем и создай `./OVERVIEW.md` согласно твоим инструкциям. Работай в текущей директории. После подтверждения превью запиши файл и верни короткий отчёт.

После возврата агента выполни проверку:

```bash
test -f ./OVERVIEW.md && echo OK || echo MISSING
```

Если `MISSING` — сообщи пользователю «Шаг 1 не завершён (OVERVIEW.md не создан). Останавливаю project-init.» и завершай без запуска следующих шагов.

## Шаг 2 — ARCHITECTURE.md

Запусти Task с `subagent_type: architecture-interviewer`. Prompt:

> Прочитай `./OVERVIEW.md`, проведи интервью по архитектуре и создай `./ARCHITECTURE.md` согласно твоим инструкциям. Работай в текущей директории.

Проверка:

```bash
test -f ./ARCHITECTURE.md && echo OK || echo MISSING
```

Если `MISSING` — стоп с сообщением «Шаг 2 не завершён (ARCHITECTURE.md не создан). Останавливаю project-init.»

## Шаг 3 — CLAUDE.md

Запусти Task с `subagent_type: claude-md-generator`. Prompt:

> Прочитай `./OVERVIEW.md` и `./ARCHITECTURE.md`, синтезируй `./CLAUDE.md` согласно твоим инструкциям. Без интервью. Покажи пользователю превью, получи подтверждение, запиши файл.

Проверка:

```bash
test -f ./CLAUDE.md && echo OK || echo MISSING
```

Если `MISSING` — стоп с сообщением «Шаг 3 не завершён (CLAUDE.md не создан). Останавливаю project-init.»

## Шаг 4 — скаффолдинг

Запусти Task с `subagent_type: project-scaffolder`. Prompt:

> Прочитай `./OVERVIEW.md`, `./ARCHITECTURE.md`, `./CLAUDE.md`. Определи тип проекта. Если KMP — создай базовый скаффолд `.claude/` согласно твоим инструкциям (с подтверждением у пользователя). Если не KMP — сообщи об ограничении v1 и выйди без ошибки.

Проверка (необязательная, только для итогового отчёта):

```bash
test -d ./.claude && echo SCAFFOLDED || echo NO_SCAFFOLD
```

`NO_SCAFFOLD` здесь не считается ошибкой — это нормально для не-KMP проектов.

## Итоговое сообщение

Выведи пользователю список созданных артефактов и подсказку:

> Готово. Созданы:
> - `./OVERVIEW.md`
> - `./ARCHITECTURE.md`
> - `./CLAUDE.md`
> - `.claude/` *(если был создан скаффолд)*
>
> Перезапусти Claude Code, чтобы подхватить новый `.claude/`.

## Важные правила

- **Не делай шагов за агентов.** Ты не задаёшь вопросы интервью и не пишешь документы сам — это работа subagent'ов.
- **Не продолжай при отказе.** Если пользователь на превью ответил «нет» — агент вернётся без файла. Это штатная остановка, просто сообщи и выходи.
- **Относительные пути.** Всегда `./OVERVIEW.md`, не абсолютные.
- **Не запускай несколько агентов параллельно** — строго последовательно.
