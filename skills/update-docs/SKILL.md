# update-docs — обновление документации проекта

Это skill вызывается командой `/dm-cc-assistant:update-docs` (или `/dm-cc-assistant:update-docs Добавил систему push-уведомлений`). Запускает `docs-updater` через Task tool.

## Твоя роль как основного Claude

Ты — диспетчер. Сам документы не анализируешь и не обновляешь — это работа subagent'а.

## Вводное сообщение

> Запускаю обновление документации. Ассистент проанализирует что изменилось, предложит правки в OVERVIEW.md, ARCHITECTURE.md, CLAUDE.md, обновит backlog и проверит открытые вопросы. Каждое изменение обсудит с тобой отдельно.

## Запуск агента

Запусти Task с `subagent_type: docs-updater`. В prompt передай:

Если `$ARGUMENTS` не пуст:

> Обнови документацию проекта после завершения работы. Описание изменений: «$ARGUMENTS». Работай в текущей директории. Обсуди каждое изменение с пользователем, получи подтверждение. Верни отчёт.

Если `$ARGUMENTS` пуст:

> Обнови документацию проекта после завершения работы. Определи что изменилось из git (diff/log) или спроси пользователя. Работай в текущей директории. Обсуди каждое изменение с пользователем, получи подтверждение. Верни отчёт.

## Итоговая подсказка

> Документация обновлена.
>
> Цикл задачи завершён. Для следующей задачи: `/dm-cc-assistant:backlog`

## Важные правила

- **Не делай шагов за агента.** Ты не анализируешь docs и не предлагаешь правки.
- **Не продолжай при отказе.** Если docs не найдены — штатная остановка с подсказкой `/dm-cc-assistant:project-init`.
- **Относительные пути.**
